# v0.2.3 安装 / 升级 Smoke 验收清单

## 前置

- [ ] 目标机器：Linux amd64 或 arm64
- [ ] 可访问 GitHub Releases
- [ ] Master 已配置 `master.env`（Telegram Token + Admin ID）

## 安装验收

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.3/scripts/install-master.sh) --yes
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.3/scripts/smoke.sh) master
```

- [ ] `qdr-master version` 输出含 `0.2.3`
- [ ] `systemctl is-active quota-dns-router-master` 为 active
- [ ] `curl -fsS http://127.0.0.1:8080/healthz` 返回 `ok`
- [ ] `curl -fsS http://127.0.0.1:8080/readyz` JSON 中 `ready=true`
- [ ] `curl -fsS http://127.0.0.1:8080/api/status` 从本机可访问（localhost 免 Token）
- [ ] `qdr-master config-check` 显示正确 DNS 服务商（cloudflare / route53）

## v0.2.3 专项

- [ ] AAAA 分组：Telegram 添加节点时接受 IPv6（如 `2001:db8::1`），拒绝 IPv4
- [ ] `qdr-agent status` 显示与 env 一致的统计模式（非硬编码 RX+TX）
- [ ] 故意错配 Agent `QDR_AGENT_TRAFFIC_MODE` 后，Master Telegram 收到一次告警
- [ ] 未配置 `QDR_STATUS_READONLY_TOKEN` 时，从非本机 IP 访问 `/api/status` 返回 401

## Agent 验收

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.3/scripts/install-agent.sh) --yes
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.3/scripts/smoke.sh) agent
```

- [ ] `qdr-agent version` 输出含 `0.2.3`
- [ ] Join 后 env 含 `QDR_AGENT_TRAFFIC_MODE`
