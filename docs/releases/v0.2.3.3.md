# v0.2.3.3 发布说明

代码审计修复：自动切换策略生效、AAAA 查询闭环、离线通知延迟、孤立代码清理。

## 修复

- **AutoSwitchEnabled**：关闭全局自动切换后，调度器不再执行 DNS 切换（此前仅 UI 显示关闭，逻辑仍切换）。
- **AAAA 记录**：节点 DNS 匹配、诊断、手动切换、Agent 安装检查等路径统一使用 `lookupGroupDNSRecord`（按 `RecordType` 查询）。
- **OfflineNotifySeconds**：离线标记（`agent_offline_seconds`）与 Telegram 通知（`offline_notify_seconds`）分离；`QDR_AGENT_OFFLINE_AFTER` / `QDR_OFFLINE_NOTIFY_AFTER` 在 Master 启动时同步到策略。
- **Agent 公网 IP**：上报时若 IP 有效且与节点配置不同，自动更新 `nodes.public_ip`。
- **切换成功文案**：`Cloudflare：已确认` 改为 `DNS：已确认`（Route53 部署不再误导）。

## 清理

- 删除 `deadcode` 检测到的 12 处未引用函数（`getInt64`/`getBool`、`ListCloudflareConfigs`、Telegram 兼容包装等）。
