# v0.2.1 安装 / 升级 Smoke 验收清单

## 前置

- [ ] 目标机器：Linux amd64 或 arm64
- [ ] 可访问 GitHub Releases
- [ ] Master 已配置 `master.env`（Telegram Token + Admin ID）

## 安装验收

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.1/scripts/install-master.sh) --yes
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.1/scripts/smoke.sh) master
```

- [ ] `qdr-master version` 输出 `0.2.1`
- [ ] `systemctl is-active quota-dns-router-master` 为 active
- [ ] `curl -fsS http://127.0.0.1:8080/healthz` 返回 `ok`
- [ ] `curl -fsS http://127.0.0.1:8080/readyz` JSON 中 `ready=true`
- [ ] 浏览器或 `curl` 访问 `/status` 可看到状态面板
- [ ] `curl -fsS http://127.0.0.1:8080/api/status` 返回 JSON

## 升级验收（自 v0.2.0）

- [ ] 重复执行 install-master 脚本不丢数据
- [ ] migration 无报错（`record_type` 列存在）
- [ ] Telegram `/status` 与 Web 面板数据一致

## Agent 验收

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.1/scripts/install-agent.sh) --yes
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.1/scripts/smoke.sh) agent
```

- [ ] `qdr-agent version` 输出 `0.2.1`
- [ ] Join 后 env 含 `QDR_AGENT_TRAFFIC_MODE`

## 安全项（可选）

- [ ] 设置 `QDR_STATUS_READONLY_TOKEN` 后，无 Token 访问 `/api/status` 返回 401
- [ ] 配置 `QDR_TELEGRAM_OBSERVER_IDS` 后，观察者无法修改 DNS

## Route53（可选）

- [ ] `QDR_DNS_PROVIDER=route53` + AWS 凭证（`AWS_ACCESS_KEY_ID` 等）
- [ ] `QDR_AWS_REGION=us-east-1`
- [ ] Master 启动无报错
