# v0.2.3

代码审查修复与运维增强版本。

## 代码审查修复

- **AAAA + IPv6 闭环**：按分组 DNS 记录类型（A / AAAA）校验节点公网 IP；Telegram 向导提示随类型变化
- **CLI DNS Provider**：`qdr-master status` / `config-check` 使用 `NewDNSProvider`，Route53 部署诊断准确
- **Agent status**：显示真实 `traffic_mode`（单向 RX / TX / 双向 RX+TX）
- **安装 URL**：随 `version.Version` 动态生成，不再硬编码 v0.1.0
- **Web 状态面板**：未配置 `QDR_STATUS_READONLY_TOKEN` 时仅 **localhost** 可访问

## 运维增强

- **Route53 语义**：`dns_provider` 持久化到 settings；诊断区分 Cloudflare Token / Route53 AWS 凭证
- **traffic_mode 告警**：上报与节点配置不一致时写入 `agent_traffic_mode_mismatch`，并 **Telegram 通知**（每 Agent 去重）
- **优雅关闭**：Master `run` 首个 worker 退出后 cancel context，等待其余 worker（10s 超时）
- **telegram-run 警告**：启动时提示仅 Bot 模式，生产请用 `qdr-master run`
- 删除孤立 `internal/system/systemd.go`

```bash

```

## 升级注意

- 公网访问 Web 状态面板需设置 `QDR_STATUS_READONLY_TOKEN`
- AAAA 分组节点须使用 **IPv6** 公网地址
- 已有 DB 中 `repo_install_url` 若为旧版，可通过 `/policy set repo=<url>` 更新
