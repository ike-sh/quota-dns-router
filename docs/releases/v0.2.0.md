# v0.2.0 发布说明

**发布日期**：2026-06-08  
**仓库**：https://github.com/ike-sh/quota-dns-router  
**完整 CHANGELOG**：见仓库根目录 `CHANGELOG.md`

## 亮点

v0.2.0 在 0.1.0 稳定基线之上，重点加强 **HTTP 安全**、**运维可观测性**、**备份恢复** 与 **DNS 类型扩展（AAAA）**，并支持 **arm64** 部署。

## 新功能

### 安全与 API
- HTTP 请求体上限 64KB，防止大包滥用
- `/api/agent/join` IP 限流（10 次/分钟）
- 结构化 HTTP 访问日志（slog）

### 健康检查
- `GET /healthz` — 存活探针
- `GET /readyz` — SQLite 就绪探针（JSON）

### 运维
- `qdr-master backup` / `restore` — SQLite 备份与恢复（恢复前自动备份当前库）
- `agent_reports` 定时清理，默认保留 30 天（`QDR_AGENT_REPORT_RETENTION_DAYS`）
- 维护窗口：Telegram 策略面板可暂停自动切换，手动切换仍可用

### DNS 与切换
- AAAA 记录基础支持：配置保存记录类型，切换按 A/AAAA 更新 Cloudflare
- Telegram DNS 向导：先选记录类型，再输入域名
- `ResolveCurrentNode` 修复：DNS IP 不匹配任何节点时不再错误回退，并通知管理员

### Agent
- Join 响应携带 `traffic_mode`，Agent 环境变量 `QDR_AGENT_TRAFFIC_MODE`
- Agent 单元测试补强

### Telegram
- 巨型 `telegram_wizard.go` 拆分为多文件，便于维护
- 多管理员：`QDR_TELEGRAM_ADMIN_IDS`（逗号分隔），通知广播全部管理员

### 发布
- GitHub Release 提供 `linux/amd64` 与 `linux/arm64` 二进制
- 安装脚本 `scripts/install-*.sh` 同步支持 arm64

## 升级指南

```bash
# 方式一：安装脚本（推荐）
bash <(curl -fsSL https://raw.githubusercontent.com/ike-sh/quota-dns-router/v0.2.0/scripts/install-master.sh)

# 方式二：仅替换二进制（已安装用户）
# 从 Release 下载对应架构 tar.gz，替换 /usr/local/bin/qdr-master 与 qdr-agent 后重启 systemd
```

**数据库**：首次启动会自动执行 migration `005_record_type.sql`，无需手动操作。

**配置新增（可选）**：
```env
QDR_TELEGRAM_ADMIN_IDS=123456789,987654321
QDR_AGENT_REPORT_RETENTION_DAYS=30
```

## 已知限制

- 仅支持 Cloudflare 作为 DNS 服务商
- 多管理员均为完全权限（只读观察者计划在后续版本）
- Master HTTP API 仍建议仅内网或反代后暴露
